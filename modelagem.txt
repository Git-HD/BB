Para entender melhor as principais atividades para criar uma modelagem de dados utilizei o exemplo de funcionários de
uma determinada empresa.
Primeiro você começa definindo suas entidades, objetos de interesse significativos. Entidades são as coisas sobre as
quais você deseja armazenar informações. Por exemplo, convém definir uma entidade chamada EMPREGADO, para funcionários
pois é necessário armazenar informações sobre todos os que trabalham na empresa. Além disso podemos definir uma entidade
chamada DEPARTAMENTO, para departamentos.

Na sequência definiremos chaves primárias (Primary Key - identificador exclusivo) para as entidades. No caso da entidade
EMPREGADO, devemos armazenar uma grande quantidade de informações. No entanto a maioria dessas informações (como data de
nascimento, gênero, endereço, data de contratação) não são uma boa opção para a chave primária. Nesse caso, regularmente
é escolhido um ID ou número exclusivo de funcionário (FUNCI_NUM) como chave primária. No caso da entidade DEPARTAMENTO,
podemos escolher um número de departamento exclusivo (DEPARTAMENTO_NUM) como chave primária.

Depois de decidir as entidades e as chaves primárias, podemos definir os relacionamentos existentes entre as entidades. Eles
são baseados nas chaves primárias. Se possuímos uma entidade para EMPREGADO e outra para DEPARTAMENTO, o relacionamento
existente são dos funcionários atribuídos a departamentos.

Depois de decidir entidades, chaves primárias e seus relacionamentos, podemos definir atributos adicionais para as entidades.
No caso da entidade EMPREGADO, podemos adicionar os seguintes atributos: Data de nascimento; Data de contratação; Endereço residencial;
Número de telefone do escritório; Gênero; Currículo. E por fim normalizamos os dados para reduzir a redundância, aumentar
a integridade e o desempenho.
